導入を開始する前に、 OnCommand Cloud Manager と Cloud Volumes ONTAP を導入するためのオプションについて理解しておくことを推奨します。

目次
  • Cloud Manager のインストール
  • Cloud Manager のセットアップ
  • Cloud Volumes ONTAP の導入

Cloud Manager のインストール

Cloud Volumes ONTAP の導入と管理には、 Cloud Manager ソフトウェアが必要です。Cloud Manager は、次のいずれかの場所に導入できます。

  • Amazon Web Services （ AWS ）

  • Microsoft Azure

  • IBM クラウド

  • 自分のネットワーク内

Cloud Manager の導入方法は、選択する場所によって異なります。

NetApp Cloud Central から Cloud Manager を導入する場合は、次の資料を参照してください。

  • AWS の概要

  • Azure の導入を開始します

その他のすべてのシナリオについては、次を参照してください。

  • Cloud Manager を Azure US Gov リージョンにインストールします

  • クラウドマネージャーを Azure ドイツ地域にインストールしています

  • Linux ホストへの Cloud Manager のインストール

Cloud Manager のセットアップ

Cloud Manager をインストールした後に、 AWS アカウントの追加、 HTTPS 証明書のインストールなど、追加のセットアップを実行することもできます。手順については、「クラウドマネージャーのセットアップ」を参照してください。

Cloud Volumes ONTAP の導入

Cloud Manager を起動して実行すると、 AWS および Microsoft Azure で Cloud Volumes ONTAP の導入を開始できます。

『 Getting Started in AWS 』と『 Getting Started in Azure 』では、 Cloud Volumes ONTAP を迅速に起動して実行するための手順を説明しています。その他のヘルプについては、次を参照してください。

  • Cloud Volumes ONTAP 9.4 でサポートされる構成

  • 構成の計画

  • AWS での Cloud Volumes ONTAP の起動

  • Azure で Cloud Volumes ONTAP を起動します
